PR #$ARGUMENTS をレビューしてください。

手順:
1. `gh pr view $ARGUMENTS` でPRのタイトル・説明・関連Issueを確認する
2. `gh pr diff $ARGUMENTS` で変更差分を取得する
3. CLAUDE.md の「PRレビュー観点(チェックリスト)」に沿ってレビューする
4. レビュー結果を以下の形式でまとめる:
   - **総評**(1〜2文)
   - **must**: マージ前に直すべき点(DV2.0設計の誤り、テスト欠落など)
   - **want**: 改善提案(命名、可読性、docs)
   - **learn**: この変更に関連して知っておくとよい知識・公式ドキュメントへのポインタ(学習リポジトリなので必ず1つ以上)
5. 内容を私に見せて確認を取ってから、`gh pr review $ARGUMENTS --comment --body "..."` でPRにコメントとして投稿する

注意:
- コードの自動修正はしない。指摘にとどめ、修正は私が行う
- 設計判断が分かれる箇所は「正解」を断定せず、選択肢とトレードオフを示す
- 指摘ゼロの場合も learn は必ず書く
