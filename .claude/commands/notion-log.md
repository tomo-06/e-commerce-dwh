マージ済みブランチの学習ログをNotionに作成してください。対象: $ARGUMENTS(PR番号またはブランチ名。省略時は直近マージされたPR)

手順:
1. `gh pr view <対象> --json title,body,url,mergedAt,headRefName` でPR情報を取得する
2. `gh pr diff <対象>` と、あればレビューコメント(`gh api` または `gh pr view --comments`)を確認する
3. CLAUDE.md の「Notionログ」テンプレートに沿って本文ドラフトを作成する:
   - **目的**: このブランチで何を達成しようとしたか(Issue/PR説明から)
   - **作業内容**: 実装・変更の要約(diffから。ファイル列挙ではなく意味のある単位で)
   - **学んだこと**: dbt Projects on Snowflake / datavault4dbt / DV2.0 に関する新しい知識
   - **詰まった点と解決**: レビュー指摘やコミット履歴から読み取れるもの+私へのヒアリング
   - **設計判断とその理由**: Hub/Link/Satの切り方、ビジネスキー選定などの判断があれば必ず記録
   - **記事ネタメモ**: Qiita/note記事化できそうなポイント
   - **次のアクション**: 次のIssue候補
4. 「詰まった点」「学んだこと」はdiffだけでは分からないことが多いので、ドラフト提示時に不足箇所を私に1〜2問だけ質問して埋める
5. 私の確認後、Notion MCPでデータベース「DV2.0学習ログ」に新規ページを作成する
   - プロパティ: ブランチ名 / Week / 日付(マージ日) / ステータス=完了 / PR URL
   - データベースが見つからない場合は作成せず、私に確認する

注意:
- ログは「未来の自分と記事の読者」が対象。コマンドの羅列ではなく、判断と理由を中心に書く
- 日本語で書く
